# إدارة المرفقات والاحتفاظ بالوثائق (Document Retention & Attachments)

## الهدف
تحديد كيفية رفع المرفقات، ومن يراها، ومدة الاحتفاظ بها، وكيف تُعامل الوثائق الحساسة مثل التقارير الطبية والفواتير وإيصالات التحويل.

---

## 1. أنواع المرفقات

| النوع | أمثلة | الحساسية |
|---|---|---|
| إثبات دفع | إيصال تحويل بنكي | متوسط |
| فاتورة | فاتورة علاج أو صيانة | متوسط إلى عالي |
| تقرير طبي | تقرير حالة علاج | عالي جداً |
| مستند هوية | هوية أو إثبات علاقة | عالي |
| مستند حوكمي | محضر اجتماع، قرار تصويت | منخفض إلى متوسط |
| مستند نزاع | أدلة أو مرفقات اعتراض | حسب المحتوى |

---

## 2. مستويات ظهور المرفقات

| المستوى | من يراه؟ |
|---|---|
| PublicToMembers | كل أعضاء الكيان |
| VisibleToParticipants | مشتركو المحفظة أو المسار |
| VisibleToCommittee | اللجنة فقط |
| VisibleToAuditor | المراجع فقط |
| HiddenSensitive | المعنيون المباشرون فقط |
| AggregatedOnly | لا يظهر الملف، يظهر ملخص فقط |

---

## 3. القاعدة الأساسية

> لا يوجد صرف بلا أثر موثق، لكن ليس كل مرفق يجب أن يظهر للجميع.

مثال:

- يظهر للأعضاء أن هناك صرف علاج بمبلغ 5,000 ريال.
- لا يظهر اسم المستفيد أو التقرير الطبي إلا للجنة والمراجع.

---

## 4. الحذف والأرشفة

لا يفضل حذف المرفقات المرتبطة بقرار مالي أو نزاع.

بدلاً من الحذف:

```text
Archived
Redacted
Hidden
Superseded
```

إذا تم حذف ملف مسموح حذفه، يجب أن يبقى أثر في سجل التدقيق.

---

## 5. Redaction / طمس البيانات

للوثائق الحساسة، يمكن توفير نسخة مطموسة:

- إخفاء رقم الهوية.
- إخفاء اسم المريض.
- إخفاء معلومات طبية غير لازمة.
- إظهار المبلغ والغرض فقط.

---

## 6. مدة الاحتفاظ

| نوع الوثيقة | مدة مقترحة |
|---|---|
| إيصالات الدفع | 5 سنوات أو حسب سياسة الكيان |
| الفواتير | 5 سنوات |
| التقارير الطبية | أقل مدة ممكنة مع حماية أعلى |
| قرارات وتصويتات | دائمة أو حتى أرشفة الكيان |
| النزاعات | حتى إغلاق النزاع + مدة احتفاظ |

يجب أن تكون مدة الاحتفاظ قابلة للضبط حسب الكيان والقانون المحلي.

---

## 7. نموذج المرفق

```text
Document {
  ownerType
  ownerId
  documentType
  fileName
  storageKey
  sensitivityLevel
  visibilityLevel
  uploadedBy
  uploadedAt
  retentionPolicyId
  redactedVersionId
  status
}
```

---

## 8. التخزين

يفضل استخدام S3-compatible storage مثل:

- MinIO في البداية.
- DigitalOcean Spaces.
- AWS S3 لاحقاً.

يجب عدم تخزين الملفات الحساسة داخل قاعدة البيانات مباشرة إلا للملفات الصغيرة جداً وبقرار واضح.

---

## 9. قواعد مهمة

1. كل مرفق مرتبط بعملية أو قرار يجب أن يبقى قابل التتبع.
2. المرفقات الطبية لا تعرض للأعضاء عمومًا.
3. حذف المرفق لا يعني حذف أثره من Audit Trail.
4. يجب دعم تصدير ملف نزاع كامل للجهة المخولة.
5. لا يسمح برفع أنواع ملفات خطرة أو غير مدعومة.

---

## MVP

- رفع إيصال دفع.
- رفع فاتورة صرف.
- تحديد مستوى خصوصية بسيط.
- منع حذف المرفقات المرتبطة بصرف مؤكد.
- سجل تدقيق للرفع والتعديل.
