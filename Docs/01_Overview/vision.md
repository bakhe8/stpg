# الرؤية (Vision)

## اسم المشروع
**CollectiveTrustOS** — نظام تشغيل للثقة الجماعية

## ما هو المشروع؟
منصة حوكمة مرنة للصناديق الاجتماعية. ليس مجرد تطبيق تحصيل وصرف، بل **محرك حوكمة اجتماعية ومالية** يسمح لأي جماعة بتصميم صندوقها حسب طبيعتها، ولكل عضو أن يشارك بالقدر والطريقة التي يثق بها.

## المشكلة التي نحلها
الصناديق الاجتماعية (صندوق عائلة، قبيلة، جيران، عمارة، حي) موجودة في كل مجتمع عربي. لكنها تعاني من مشاكل متكررة:

| المشكلة | السؤال الذي يطرحه العضو |
|---|---|
| غياب الشفافية | كيف أثق أن المال سيذهب للمكان الصحيح؟ |
| الإجبار | كيف لا أُجبر على بنود لا أريدها؟ |
| غياب الحوكمة | كيف أشارك تحت حوكمة أقبلها؟ |
| غياب الوضوح | كيف أعرف حقوقي والتزاماتي؟ |
| الخلافات | كيف أعترض بدون قطيعة؟ |
| خلط الأموال | كيف لا تختلط أموال العائلة والعمارة والحي؟ |
| تداخل العضويات | كيف نرى التداخل بدل أن ندفن المشكلة؟ |
| اختلاف الثقة | كيف نجعل الاختلاف في طريقة الإدارة قابلاً للتعايش؟ |

## الفكرة الجوهرية
> **لا يوجد صندوق واحد يناسب الجميع. لذلك لا نبني صندوقاً واحداً، بل نبني منصة تجعل كل جماعة تصمم صندوقها، وكل عضو يدخل بالقدر والطريقة التي يثق بها، وكل قرار يبقى موثقاً وقابلاً للمراجعة.**

## المبادئ التأسيسية

### 1. الواقع الاجتماعي معقد — والنظام لا ينكره
- الناس مختلفون في مستوى الثقة.
- القرابة تتداخل مع السكن.
- المال حساس.
- التصويت ليس دائماً كافياً.
- القرار الفردي أحياناً مفيد وأحياناً خطير.
- الخصوصية مطلوبة، والشفافية مطلوبة.
- بعض الناس يريد المشاركة، لكن بشرط.

### 2. زر بسيط فوق محرك عميق
النظام من الداخل معقد، لكن الواجهة لا يجب أن تكون معقدة. العضو العادي لا يرى نموذج البيانات، بل يرى:
- هل يناسبك هذا الاشتراك؟
- **نعم / لا / أشارك بشرط**

والنظام في الخلفية يترجم ذلك إلى قواعد.

### 3. التعقيد يُفتح تدريجياً
النظام لا يبدأ من المستوى الأعلى. يفتح التعقيد فقط عندما يحتاجه المستخدم:
- **المستوى البسيط:** كيان + محفظة واحدة + مسؤول واحد + اشتراك ثابت.
- **المستوى المنظم:** عدة محافظ + لجنة + تصويت + اعتراضات + بنود صرف.
- **المستوى المتقدم:** عدة كيانات + مسارات حوكمة + علاقات بين كيانات + محافظ مشتركة + نزاعات.

## أنواع الصناديق المدعومة

| نوع الصندوق | الأعضاء | أمثلة الصرف |
|---|---|---|
| صندوق العائلة | أفراد عائلة | علاج، وفاة، زواج، مساعدة قريب |
| صندوق القبيلة | أبناء قبيلة أو فخذ | ديات، مساعدات كبيرة، مناسبات عامة |
| صندوق الجيران | سكان حي / شارع / عمارة | صيانة مشتركة، حراسة، إنارة، نظافة، مساعدة جار |
| صندوق الطوارئ | أي مجموعة | مرض، حادث، وفاة، أزمة مفاجئة |
| صندوق الجماعة | اسم عام | يشمل أي مجموعة حسب الاتفاق |

## ما يميز المشروع
1. **مسارات حوكمة متعددة** داخل نفس المحفظة — يتفق الأعضاء على الهدف لكن يختلفون في طريقة الثقة، فيُفصل القرار دون كسر الهدف.
2. **محرك توافق** يربط حقوق العضو بدرجة توافق شروطه مع المسار.
3. **محرك علاقات** يدير التداخل بين الكيانات ويمنع خلط الأموال.
4. **شفافية قابلة للضبط** — نخفي الإنسان عند الحاجة، ولا نخفي المال.
5. **سجل تدقيق غير قابل للعبث** — كل قرار موثق وقابل للمراجعة.

## الخلاصة
المشروع ليس تطبيقاً صغيراً. هو **منصة حوكمة مرنة** للصناديق الاجتماعية، قلبها محرك توافق ومحرك علاقات وطبقة شفافية ومساءلة، تجعل التعقيد واضحاً بدل أن يكون سبباً للفشل.
