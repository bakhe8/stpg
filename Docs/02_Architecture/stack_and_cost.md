# الاستاك والتكلفة الاقتصادية (Stack & Economics)

تم اختيار الاستاك التقني بحيث يوازن بين قوة الأداء، ووضوح المعمارية، وتكلفة التشغيل المنخفضة في البداية، مع قابلية التوسع لاحقاً.

## 1. الاستاك الأساسي (المجاني والمفتوح المصدر)
- **Frontend:** Next.js + TypeScript (مجاني ومفتوح).
- **Backend:** NestJS + TypeScript (مجاني ومفتوح، وبنية Modular ممتازة).
- **Mobile:** React Native / Expo (مجاني ومفتوح، يغطي iOS و Android بكود واحد).
- **Database:** PostgreSQL (مفتوح المصدر ومجاني).
- **ORM:** Prisma (مجاني).
- **Queue/Cache:** Redis + BullMQ (مفتوح المصدر).

## 2. خيارات النشر الأولية الأرخص (Initial Deployment)
لتقليل التكلفة في النسخ 1 و 2، يتم النشر باستخدام **Docker Containers** على **VPS** (مثل DigitalOcean أو Hetzner) بتكلفة لا تتجاوز 50-100 دولار شهرياً. 
- الاستغناء حالياً عن الخدمات المدارة باهظة الثمن (Managed Services) طالما أن حجم البيانات ليس ضخماً جداً في السنة الأولى.
- استخدام S3-compatible storage رخيص مثل MinIO أو DigitalOcean Spaces لحفظ المرفقات الطبية والفواتير.

## 3. تقنيات مؤجلة للمراحل المتقدمة (Delayed for Scale)
لماذا لا نبدأ بـ Kubernetes أو Microservices أو Temporal Cloud؟ لأنها ترفع تعقيد التشغيل وتكلفة الخوادم أضعافاً مضاعفة قبل إثبات المنتج.
- **Kubernetes (K8s):** سيتم تأجيله حتى نحتاج لتوسع أفقي ضخم.
- **Microservices:** نبدأ بـ Modular Monolith، ونفصل الخدمات (مثل الإشعارات أو الدفع) فقط عندما تصبح عنق زجاجة.
- **Temporal:** محرك مسارات العمل (Workflow Engine) قوي جداً للنزاعات والمراجعات طويلة الأمد، لكنه يستهلك موارد عالية. سيتم إضافته في النسخة 3 أو 4.
- **OpenSearch:** للبحث المتقدم في السجلات، سنكتفي بالبحث المدمج في PostgreSQL (Full Text Search) في البداية.
