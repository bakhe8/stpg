# 08 - Phase D Parity Pack

## حالة الوثيقة

هذه وثيقة إغلاق `D-009` داخل Phase D.

الحالة: منفذة ومتحقق منها.

التاريخ: 2026-07-01.

## الهدف

إثبات أن مسار إنشاء الصندوق/الحملة الجديد لا يختصر القدرات التي بُني عليها النظام قبل التفكير في جعله المسار الافتراضي.

هذه الحزمة لا تجعل المسار الجديد default. هي بوابة تحقق قبل `D-010` و`D-011`.

## ما أضيف

أضيف اختبار backend مستقل:

- `backend/src/entities/phase-d-parity.spec.ts`

وأضيف أمر تشغيل مباشر:

- `npm run test:phase-d:parity`

الأمر يشغل:

- `entities/phase-d-parity.spec.ts`
- `entities/entities.service.spec.ts`

## نطاق التحقق

الـ parity pack يغطي القدرات التالية:

| القدرة | دليل التغطية |
|---|---|
| التصويت ونماذج القرار | `governancePathTypes`, `decisionTypes`, `decisionStatuses` داخل seed stories. |
| اللجان والمراجعة المفوضة | `COMMITTEE`, `BOARD`, ومستويات خصوصية اللجنة. |
| المال والدفعات | أنواع منفعة المحافظ، حالات الدفعات المستحقة، وحالات سجلات الدفع. |
| الصرف | قرارات `DISBURSE_FUNDS` وحالات طلبات الصرف. |
| الاشتراكات | `ACTIVE`, `CONDITIONAL`, `SUSPENDED`, `EXITED`, `SUPPORTER_ONLY`. |
| النزاعات والاعتراضات | حالات appeal/dispute ومستويات خصوصية المستندات. |
| الحملات | `CAMPAIGN`, حالة `READ_ONLY`, مسار `DONATION_ONLY`, وقرار `CLOSE_WALLET`. |
| المحافظ المتعددة والعلاقات | علاقات الكيانات، علاقات المحافظ، حالات الموافقة، وحقوق العلاقة. |
| إنشاء الحملة | اختبار خدمة مباشر يثبت أن الحملة تنشأ كـ `CAMPAIGN` مرتبطة بصندوق أب مع `isCampaign`, `campaignEndsAt`, ledger, founder membership, وaudit. |
| إنشاء الصندوق بالقالب | اختبار خدمة موجود وموسع يثبت أن القالب ينشئ policy, wallet, path, ledger accounts, spending items, وaudit. |

## حدود D-009

D-009 لا تفحص الواجهة بصريا. هذا يبقى في `D-010`.

لا يتم في D-009:

- لا جعل المسار الجديد default.
- لا حذف legacy flow.
- لا تغيير routes.
- لا تغيير schema.
- لا تغيير seed data.

## أثر المنتج

هذا يثبت أن التبسيط في الواجهة لم يحول الصندوق إلى منتج محدود.

المؤسس يمكن أن يبدأ من مسار بسيط، لكن القدرات العميقة ما زالت مثبتة في النظام:

- تصويت.
- لجان.
- محافظ ومسارات.
- صرف.
- اشتراكات.
- نزاعات واعتراضات.
- تدقيق.
- حملات.
- علاقات بين صناديق ومحافظ.

## المتبقي بعد D-009

قبل أي default switch:

1. `D-010`: UX smoke test للعلمين `flag off` و`flag on`.
2. `D-011`: قرار إداري/منتجي بجعل المسار الجديد default أو إبقائه خلف flag فترة أطول.
