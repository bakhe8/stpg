# 09 - Phase D UX Smoke Tests

## حالة الوثيقة

هذه وثيقة إغلاق `D-010` داخل Phase D.

الحالة: منفذة ومتحقق منها.

التاريخ: 2026-07-01.

## الهدف

إثبات أن مسار إنشاء الصندوق/الحملة الجديد يمكن تشغيله خلف feature flag بدون كسر المسار القديم.

هذه الحزمة لا تجعل المسار الجديد default. هي فحص واجهة قبل قرار `D-011`.

## ما أضيف

أضيف اختبار Playwright مستقل:

- `frontend/scripts/phase-d-create-flow-smoke.spec.cjs`

وأضيف أمر تشغيل مباشر:

- `npm run test:phase-d:create-flow`

## نطاق التحقق

الاختبار يشغل Next dev server مؤقتا مرتين:

| الحالة | المنفذ الافتراضي | قيمة العلم | المطلوب إثباته |
|---|---:|---|---|
| Legacy flow | `3100` | `NEXT_PUBLIC_ENABLE_FUND_CREATE_FLOW=false` | شاشة الإنشاء القديمة تظهر تصنيفات عائلة/عمارة/حي/قبيلة/حملة مؤقتة، ولا تظهر شاشة صندوق/حملة الجديدة. |
| New flow | `3333` | `NEXT_PUBLIC_ENABLE_FUND_CREATE_FLOW=true` | تظهر شاشة الاختيار بين صندوق وحملة، ويبقى زر استخدام النموذج القديم، وتظهر شاشة بدء الصندوق وخريطة تشغيل الحملة. |
| Default flow | `3100` | فارغة/غير `false` | تظهر شاشة صندوق/حملة الجديدة كافتراضي بعد D-011. |

يمكن تغيير المنافذ عند الحاجة:

```bash
PHASE_D_LEGACY_PORT=3100 PHASE_D_NEW_PORT=3333 npm run test:phase-d:create-flow
```

## ما يفحصه الاختبار

في حالة العلم مطفأ:

- تسجيل دخول تطويري بمستخدم seed.
- فتح `/entities/new`.
- تجاوز شاشة التنبيه القانونية إن ظهرت.
- التأكد من ظهور `اختر نوع الكيان`.
- التأكد من ظهور `عائلة`، `عمارة`، `حي`، `قبيلة`، و`حملة مؤقتة`.
- التأكد من عدم ظهور سؤال `ماذا تريد أن تنشئ؟`.

في حالة العلم مفعّل:

- تسجيل دخول تطويري بمستخدم seed.
- فتح `/entities/new`.
- تجاوز شاشة التنبيه القانونية إن ظهرت.
- التأكد من ظهور سؤال `ماذا تريد أن تنشئ؟`.
- التأكد من ظهور خياري `صندوق` و`حملة`.
- التأكد من بقاء زر `استخدام النموذج القديم`.
- التأكد من أن شاشة الصندوق تعرض `طريقة بدء الصندوق` و`ابدأ فارغاً` والقوالب التشغيلية المبسطة.
- التأكد من أن شاشة الصندوق الجديدة لا تعرض عنوان القوالب القديم `اختر قالباً جاهزاً`.
- التأكد من أن شاشة الحملة تعرض `خريطة تشغيل الحملة` و`الصندوق الأب` و`بعد الإنشاء` وزر `إنشاء الحملة`.

## حدود D-010

D-010 لا تنشئ صندوقا أو حملة فعليا. الهدف فحص المسار المرئي لا اختبار كتابة البيانات.

لا يتم في D-010:

- لا جعل المسار الجديد default.
- لا حذف legacy flow.
- لا تغيير routes.
- لا تغيير schema.
- لا تغيير backend contract.

## دليل التشغيل

الأمر:

```bash
npm run test:phase-d:create-flow
```

يعتمد على:

- backend API يعمل على `http://localhost:3001/api`.
- seed users موجودين.
- `NEXT_PUBLIC_ENABLE_DEV_LOGIN=true` داخل بيئة السيرفر التي يشغلها الاختبار.
- المنافذ `3100` و`3333` متاحة ومسموحة في CORS.

الاختبار يحفظ لقطات الشاشة وملفات summary في مجلد مؤقت خارج المستودع:

- `STGP_PHASE_D_SMOKE_OUT_DIR` إذا تم تحديده.
- أو مجلد داخل temp باسم `stgp-phase-d-create-flow-smoke-*`.

## أثر المنتج

أصبح لدينا دليل واجهي قابل للإعادة أن:

- إطفاء العلم يبقي تجربة الإنشاء القديمة كما هي.
- تفعيل العلم يفتح تجربة الصندوق/الحملة الجديدة دون حذف مسار الرجوع.
- خيارات الصندوق الجديدة تعرض القوالب كنقاط بداية تشغيلية لا كتفاصيل تقنية.
- الحملة تظهر كمسار مستقل مرتبط بصندوق، لا كنوع صندوق عادي.

## المتبقي بعد D-010

تم تنفيذ `D-011` لاحقا في:

- `Docs/09_Improvement/10_PHASE_D_DEFAULT_SWITCH.md`

وأصبح المسار الجديد default مع بقاء `NEXT_PUBLIC_ENABLE_FUND_CREATE_FLOW=false` كمسار rollback.
