# 07 - Phase D Parallel Create Flow

## حالة الوثيقة

هذه هي حزمة Phase D داخل `Docs/09_Improvement`.

الحالة الحالية: بدأ التنفيذ بأول شريحة آمنة خلف feature flag، ولم يصبح المسار الجديد default بعد.

## الهدف

بناء مسار إنشاء جديد للمستخدم العادي يفصل بين:

- صندوق دائم.
- حملة مؤقتة مرتبطة بصندوق قائم.

مع بقاء شاشة الإنشاء القديمة متاحة كمسار رجوع إلى أن يثبت parity pack.

## حدود Phase D

Phase D لا تعني حذف القديم.

لا نفعل في هذه المرحلة:

- لا نحذف `/entities/new`.
- لا نحذف `EntityType`.
- لا نغير `/entities` إلى `/funds`.
- لا نجعل المسار الجديد default قبل التحقق.
- لا ننشئ campaign مستقلة بلا parent fund.

## Feature Flag

العلم المستخدم:

- `NEXT_PUBLIC_ENABLE_FUND_CREATE_FLOW`

السلوك:

| القيمة | النتيجة |
|---|---|
| `false` أو غير موجود | شاشة الإنشاء القديمة تبقى كما هي. |
| `true` | تظهر شاشة اختيار `صندوق` أو `حملة` داخل `/entities/new`. |

## ما تم في أول شريحة

| التذكرة | الحالة | التفاصيل |
|---|---|---|
| `D-001` | منفذة | إضافة feature flag في frontend. |
| `D-002` | منفذة | شاشة دخول جديدة تفصل صندوق/حملة عند تفعيل flag. |
| `D-003` | منفذة جزئيا | إنشاء صندوق من المسار الجديد يستخدم `type = COMMUNITY` داخليا، ويحفظ `profileKey/profileLabel` اختياريا. |
| `D-004` | منفذة جزئيا | إنشاء حملة يستخدم endpoint الحالي `POST /entities/:id/campaigns` ويتطلب اختيار صندوق قائم. |
| `D-005` | منفذة | زر رجوع إلى النموذج القديم داخل المسار الجديد. |

## ما لم يكتمل بعد

| التذكرة | المطلوب |
|---|---|
| `D-006` | بناء setup checklist بعد إنشاء صندوق فارغ. |
| `D-007` | تحسين شاشة الحملة بتفاصيل مالية/تشغيلية خاصة بالحملة. |
| `D-008` | إخفاء تفاصيل القوالب التقنية أكثر وإظهارها كاختيارات تشغيلية واضحة. |
| `D-009` | بناء parity pack يغطي القدرات الأساسية قبل جعل المسار الجديد default. |
| `D-010` | إضافة UX smoke test للعلمين: flag off وflag on. |
| `D-011` | قرار جعل المسار الجديد default بعد نتائج parity. |

## قرارات المنتج المثبتة

- الصندوق الجديد يبدأ كـ `COMMUNITY` داخليا إذا لم يكن هناك سبب تشغيلي لاستخدام نوع قديم.
- عائلة/عمارة/حي/قبيلة أصبحت profile اختياري، لا تقود السلوك.
- الحملة لا تنشأ منفصلة في المرحلة الأولى؛ يجب ربطها بصندوق قائم.
- النموذج القديم يبقى متاحا للفريق والمقارنة.

## ضمان عدم خسارة القدرات

المسار الجديد لا يختصر الباكند:

- اختيار القالب ما زال يطبق Phase B ويُنشئ المحافظ والمسارات.
- الصندوق الفارغ يبقى قابلا للتوسيع لاحقا بالمحافظ والمسارات والقواعد.
- الإعدادات المتقدمة تبقى في مسارها الحالي وتخضع لصلاحية Phase C.
- الحملة تستخدم lifecycle الحالي: parent fund، `isCampaign`, و`campaignEndsAt`.

## شروط إغلاق Phase D

لا تعتبر Phase D مغلقة إلا إذا تحقق التالي:

1. flag off يحافظ على legacy flow بدون تغير.
2. flag on يسمح بإنشاء صندوق من المسار الجديد.
3. flag on يسمح بإنشاء حملة تابعة لصندوق قائم.
4. صندوق بلا قالب يظهر له setup checklist واضح.
5. صندوق بقالب ينشئ نفس البنية التشغيلية المثبتة في Phase B.
6. لا تظهر الحملة كنوع صندوق عادي في المسار الجديد.
7. parity pack يغطي التصويت، اللجان، المال، التدقيق، الصرف، الاشتراكات، النزاعات، الحملات، والمحافظ المتعددة.

## التحقق المطلوب لهذه الشريحة

- `npm run build` في `frontend`.
- `npm run lint` في `frontend`.

Backend لا يتغير في أول شريحة لأن endpoint الحملة موجود مسبقا.
