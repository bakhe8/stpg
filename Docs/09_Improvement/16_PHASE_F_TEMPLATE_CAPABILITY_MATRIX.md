## القرار المختصر

القوالب في تجربة إنشاء الصندوق هي نقاط بداية تشغيلية فقط. لا يوجد قالب يقفل الصندوق داخل نموذج واحد، ولا يوجد قالب يمنع المؤسس لاحقا من إضافة محافظ أو مسارات أو لجان أو قواعد أو علاقات أو عمليات مالية أوسع.

هذا القرار يحفظ التوازن الذي اتفقنا عليه:

- المستخدم يرى "صندوق" و"حملة"، لا يرى "كيان".
- صندوق عائلة، صندوق عمارة، صندوق حي، أو صندوق مخصص كلها تخصيصات وصفية وليست نماذج تقنية منفصلة.
- القالب يختصر بداية التشغيل، ولا يحذف العمق الموجود في النظام.
- الحملة تبقى مسارا منفصلا لأنها lifecycle مؤقت ومرتبط بصندوق أب.

## مصادر الحقيقة

| المصدر | ما يثبته |
|---|---|
| `backend/prisma/seed.ts` | القوالب النشطة seeded حاليا: صندوق مخصص، صندوق تكافل، صندوق خدمات مشتركة، صندوق داعمين فقط. |
| `backend/src/entities/entity-template-schema.ts` | schema القالب يحول `defaultPolicy/defaultWallets/defaultPaths` إلى إعدادات تشغيلية صالحة، ولا يعرف حقولا تقفل القدرات اللاحقة. |
| `backend/src/entities/entities.service.ts` | `createEntity` يحفظ `templateId` ويطبق المحافظ والمسارات الافتراضية فقط؛ و`createCampaign` ينشئ حملة مرتبطة بدون قالب. |
| `backend/prisma/schema.prisma` | `templateId` موثق كمرجع تاريخي لا يقيد الصندوق، و`profileKey/profileLabel` وصف اختياري لا يقود السلوك التشغيلي. |
| `frontend/src/app/(main)/entities/new/page.tsx` | الواجهة تعرض `ابدأ فارغا` والقوالب كاختيارات تشغيلية مبسطة، وتفصل خيار الصندوق عن الحملة. |
| `backend/src/entities/phase-d-parity.spec.ts` | يغطي القدرات التي يجب أن تبقى موجودة بعد default switch. |

## مصفوفة القوالب

| خيار البداية | ما ينشأ مباشرة | ما يستطيع المؤسس إضافته لاحقا | قدرات لا يجوز أن يحجبها الخيار | نوع المنفعة عند البداية |
|---|---|---|---|---|
| `ابدأ فارغا` | صندوق `COMMUNITY` مع policy فارغة، ledger account، وعضوية مؤسس. لا تنشأ محافظ أو مسارات. | كل شيء من داخل الصندوق: محافظ، مسارات حوكمة، لجان، اشتراكات، قواعد، أعضاء، علاقات، مستندات، صرف، تدقيق. | لا يجوز اعتباره صندوقا ناقصا أو محدودا. هو اختيار واع لتأجيل الإعداد. | لا يوجد نوع منفعة حتى ينشئ المؤسس محفظة. |
| صندوق مخصص `CUSTOM_FUND` | محفظة رئيسية `SEPARABLE`، مسار قرار أساسي `BOARD`، سياسات قرار ولجنة، modules مقترحة: payments/decisions/audit. | إضافة محفظة مشتركة `SHARED`، إضافة مسارات `PUBLIC_VOTE` أو `COMMITTEE` أو `EMERGENCY_FAST`، إضافة بنود صرف، علاقات، لجان، قواعد، وثائق. | لا يحجب التكافل، الخدمات المشتركة، الداعمين، المحافظ المتعددة، التصويت العام، اللجان، الصرف، التدقيق، أو النزاعات. | `SEPARABLE` |
| صندوق تكافل `MUTUAL_AID` | محفظة تكافل `SEPARABLE`، مسار لجنة تكافل `COMMITTEE`، بند مساعدة علاجية، مستندات وخصوصية أعلى. | إضافة محفظة خدمات مشتركة، إضافة مسار تصويت عام، إضافة حملة مرتبطة، إضافة قواعد أهلية أو اعتراضات، إضافة علاقات دعم مع صناديق أخرى. | لا يحجب الخدمات المشتركة، التبرعات، المحافظ المشتركة، العلاقات، مراجعة المراجع، أو المسارات غير الطبية. | `SEPARABLE` |
| صندوق خدمات مشتركة `SHARED_SERVICES` | محفظة خدمات مشتركة `SHARED`، مسار لجنة خدمات `COMMITTEE`، بند صيانة وخدمات، شفافية أعلى للأعضاء. | إضافة محفظة تكافل `SEPARABLE`، إضافة بنود دعم فردية، إضافة مسار طوارئ، إضافة لجان متعددة، إضافة علاقات أو حملات. | لا يحجب المساعدات الفردية، الداعمين، التصويت، الاعتراضات، الصرف، الاشتراكات، أو المحافظ القابلة للفصل. | `SHARED` |
| صندوق داعمين فقط `SUPPORTER_ONLY` | محفظة دعم `SEPARABLE`، مسار `DONATION_ONLY`، قرار فردي بسقف، بلا حق استفادة تلقائي للدافعين من هذه المحفظة. | إضافة محفظة أخرى تمنح حقوق استفادة، إضافة مسار لجنة، إضافة بنود صرف بحوكمة مختلفة، إضافة حملة أو علاقة دعم، إضافة قواعد تدقيق. | لا يحجب التحول لاحقا إلى صندوق أوسع؛ لكنه لا يعطي حق استفادة تلقائي من محفظة الدعم نفسها. | `SEPARABLE` مع معنى داعمين فقط على محفظة البداية. |
| حملة مرتبطة بصندوق أب | سجل حملة `CAMPAIGN` تحت صندوق أب، `isCampaign=true`، policy وledger وعضوية مؤسس. لا تستخدم `templateId`. | تجهيز محفظة/مسار الحملة بعد الإنشاء حسب الحاجة، وربطها بتشغيل الصندوق الأب والمدة والحساب البنكي. | لا تتحول إلى بديل عن الصندوق الدائم. لا تحجب التدقيق أو المستندات أو الصرف أو الإغلاق/الأرشفة. | لا يوجد نوع منفعة حتى تضاف محفظة للحملة. |

## القدرات التي يجب أن تبقى مفتوحة بعد أي قالب

أي صندوق دائم، سواء بدأ فارغا أو من قالب، يجب أن يستطيع الوصول لاحقا إلى هذه القدرات:

| المجال | القدرات المطلوبة |
|---|---|
| الحوكمة والتصويت | `BOARD`, `COMMITTEE`, `PUBLIC_VOTE`, `INDIVIDUAL_WITH_CAP`, `DONATION_ONLY`, `EMERGENCY_FAST` وأنواع القرار الأساسية مثل إنشاء محفظة/مسار، الصرف، النقل، الإغلاق. |
| اللجان والتفويض | إنشاء لجان، ربط لجنة بمسار، تفويض عضو للإعدادات المتقدمة، فصل صلاحية التشغيل المالي عن صلاحية السياسة. |
| المال والمحافظ | محافظ متعددة، `SHARED` و`SEPARABLE`، سياسات اشتراك، رسوم/تبرعات/مساهمات، مستحقات، سجلات دفع، ledger accounts. |
| الصرف والتدقيق | طلبات صرف، موافقة/رفض/تنفيذ، مستندات، audit logs، مراجعة المراجع، تقارير صحة الصندوق. |
| العضوية والمشاركة | دعوات، طلبات انضمام، اشتراكات، حالات `ACTIVE/CONDITIONAL/SUSPENDED/EXITED/SUPPORTER_ONLY`. |
| النزاعات والاعتراضات | Appeals، disputes، خصوصية المستندات، التصعيد، الإغلاق. |
| العلاقات والتوسع | علاقات بين صناديق، علاقات بين محافظ، دعم بدون تصويت، دعم مع إشراف، مشاركة تقارير. |
| الحملات | حملات مرتبطة بصندوق أب، مدة/إغلاق، تشغيل مالي مستقل عند الحاجة، مع بقاء الصندوق الأصل هو الحاوية الدائمة. |

## ملاحظات تنفيذية مهمة

- `enabledModules` يبدأ من القالب ويظهر كـ metadata/عرض، ولا يستخدم حاليا كقيد يمنع API أو الصفحة من العمل.
- `profileKey/profileLabel` اختيار وصفي للمستخدم مثل عائلة أو عمارة أو حي، وليس مصدر سلوك تشغيلي.
- `Entity.type` يبقى داخليا للتوافق. مسار إنشاء الصندوق الجديد يستخدم `COMMUNITY` للصندوق الدائم، و`CAMPAIGN` للحملة.
- لا نضيف قالبا جديدا إذا كان الهدف مجرد تسمية اجتماعية؛ التسمية الاجتماعية مكانها profile اختياري أو اسم الصندوق.
- لا نستخدم القالب كبديل عن شاشة الإعدادات المتقدمة. القالب يبدأ الوضع، والإعدادات المتقدمة تعدل الوضع.

## مخاطر التنفيذ وكيف عولجت

| الخطر | المعالجة |
|---|---|
| فهم القالب كقيود دائمة | وثقنا أن `templateId` مرجع تاريخي وأن القوالب لا تغلق القدرات. |
| خلط profile الاجتماعي مع التشغيل | بقي profile اختياريا، والسلوك يأتي من المحافظ والمسارات والسياسات. |
| حصر صندوق الداعمين كداعم فقط للأبد | المصفوفة توضح أن محفظة البداية داعمين فقط، لكن الصندوق يستطيع إضافة محافظ ومسارات أخرى. |
| اعتبار `enabledModules` بوابة صلاحيات | فحص الكود أثبت أنه metadata فقط حاليا، ويجب ألا يتحول إلى gate بدون قرار منتج مستقل. |
| خلط الحملة مع القالب | الحملة موثقة كمسار lifecycle مستقل، لا كقالب صندوق. |

## معيار الإغلاق

تم إغلاق F-004 لأن:

- توجد مصفوفة واضحة للفريق تغطي كل خيارات البداية.
- `ابدأ فارغا` موثق كخيار تشغيل صحيح لا كحالة ناقصة.
- كل قالب موثق بما ينشئه مباشرة وما لا يجوز أن يحجبه لاحقا.
- `SHARED` و`SEPARABLE` مثبتان في مواضعهما.
- الحملة موثقة خارج قوالب الصندوق لأنها مرتبطة بصندوق أب ولها lifecycle مختلف.
- لم نغير schema أو routes أو API.
