# القرارات والتصويت (Governance Paths & Voting)

## المبدأ
كل قرار في النظام يكون **كائناً مستقلاً** قابلاً للتتبع والتدقيق والاعتراض. لا يوجد قرار شفهي أو غير موثق.

---

## نموذج القرار (Decision Model)

```
Decision {
  subjectType    -- نوع الموضوع (محفظة، مسار، صرف، عضوية)
  subjectId      -- معرف الموضوع
  decisionType   -- نوع القرار
  requiredApproval -- نوع الموافقة المطلوبة
  votersScope    -- من يحق له التصويت
  result         -- النتيجة (مقبول / مرفوض / معلق)
  status         -- الحالة (مفتوح / مغلق / معترض عليه)
  auditTrail     -- سجل الأحداث
}
```

## أنواع القرارات

| القرار | مثال |
|---|---|
| إنشاء محفظة | فتح محفظة طوارئ جديدة |
| إنشاء مسار | إضافة مسار "قرار فردي" لمحفظة الطوارئ |
| صرف مبلغ | الموافقة على صرف 3,000 ريال لعلاج عاجل |
| تعديل اشتراك | تغيير مبلغ الاشتراك الشهري |
| تعديل حوكمة | تحويل القرار من لجنة إلى تصويت عام |
| نقل رصيد | تحويل فائض من مسار إلى آخر |
| قبول عضو | الموافقة على عضوية جديدة |
| فتح نزاع | فتح قضية نزاع رسمية |
| إغلاق محفظة | إيقاف محفظة وتوزيع رصيدها |
| دمج مسارات | توحيد مسارين داخل محفظة واحدة |

## أنواع التصويت

| نوع التصويت | حالة الاستخدام | ملاحظات |
|---|---|---|
| صوت لكل عضو | قرارات عامة | الأكثر شيوعاً |
| صوت لكل أسرة | عائلات | لمنع هيمنة العائلات الكبيرة |
| صوت لكل مشترك في المسار | قرارات المسار | فقط من يدفع يصوت |
| صوت حسب المساهمة | مشاريع خاصة | يُستخدم بحذر |
| أغلبية بسيطة (50%+1) | قرارات عادية | |
| أغلبية الثلثين | تعديل لائحة أو دستور | للقرارات الثقيلة |
| تصويت سري | قرارات حساسة | لحماية المصوّتين |
| موافقة لجنة | قرارات تشغيلية | اللجنة وحدها تقرر |
| قرار فردي بسقف | خدمات بسيطة | المدير يقرر حتى مبلغ محدد |
| قرار عاجل ثم مراجعة | طوارئ | ينفذ فوراً ويُراجع لاحقاً |

---

## قواعد التصويت

1. **لا يصوت العضو في مسار ليس مشتركاً فيه.** مثلاً: العضو في مسار مجلس الإدارة لا يصوت في قرارات مسار القرار الفردي.

2. **نتيجة التصويت تُحفظ وتُوثق** — سواء كانت علنية أو سرية.

3. **لكل قرار مدة زمنية** — إذا انتهت المدة دون نصاب، يُعاد أو يُغلق حسب سياسة الكيان.

4. **القرارات التي تؤثر على شروط العضو** (مثل تغيير الحوكمة) تتطلب **إعادة موافقة** الأعضاء المتأثرين.

> لتفاصيل حماية العضو عند تغيير الشروط، انظر: [member_participation_contract.md](member_participation_contract.md)
