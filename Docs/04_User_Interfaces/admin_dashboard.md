# لوحة تحكم المؤسس والمدير (Admin Dashboard)

> **نطاق هذه الوثيقة: Tenant Layer فقط.**
> هذه لوحة مؤسس الكيان أو مديره — لا علاقة لها بلوحة مشرف المنصة (Platform Dashboard).
> للفرق الكامل بين المحورين: [platform_vs_tenant_roles.md](../05_Rules_and_Governance/platform_vs_tenant_roles.md)

## الفلسفة
المؤسس أو المدير يحتاج أدوات لإنشاء الصندوق وإدارته، لكن بتدرج. لا يظهر له كل الخيارات دفعة واحدة، بل يُقاد بمعالج (Wizard) خطوة بخطوة.

> **ملاحظة على معالج الإنشاء:** قبل دخول هذا المعالج، يمر المستخدم بخطوتين على مستوى المنصة: التحقق من اشتراكه، ثم تحديد نوع الصندوق مع التحذير القانوني المناسب. انظر: [LEGAL_AND_COMPLIANCE_BOUNDARIES.md](../06_Optional_Policy/LEGAL_AND_COMPLIANCE_BOUNDARIES.md)

## الشاشات الأساسية

### 1. معالج إنشاء الكيان (Creation Wizard)
خطوات إنشاء صندوق جديد:

| الخطوة | الوصف |
|---|---|
| 1. اختر النوع | عائلة / عمارة / حي / قبيلة / جيران / حملة مؤقتة |
| 2. حدد الاسم والوصف | معلومات أساسية عن الصندوق |
| 3. اختر نموذج الحوكمة العام | مجلس إدارة / لجنة / تصويت عام / مختلط |
| 4. أضف محافظ | طوارئ / خدمات / مناسبات / مشاريع |
| 5. اسمح أو لا تسمح بالمسارات | هل يُسمح بتعدد مسارات الحوكمة داخل المحفظة؟ |
| 6. افتح استطلاع شروط الأعضاء | جمع تفضيلات الأعضاء قبل التفعيل |
| 7. راجع وفعّل | مراجعة نهائية ثم تفعيل النموذج الأنسب |

### 2. إدارة المحافظ
- إنشاء محفظة جديدة
- تعديل قواعد الاشتراك والاستفادة
- فتح / إغلاق / أرشفة محفظة
- ربط محفظة بكيان آخر (محفظة مشتركة)

### 3. مسارات الحوكمة
- إنشاء مسار جديد داخل محفظة
- تحديد أعضاء المسار وقواعد قراره
- تحليل التوافق: كم عضو سينضم لكل مسار؟
- تفعيل / تعليق / دمج مسارات

### 4. إدارة الأعضاء
- قبول / رفض طلبات العضوية
- مراجعة حالات الأعضاء (Active / Suspended / Conditional)
- نقل عضو بين المسارات (بموافقته)
- عرض إجمالي التزامات العضو عبر كل الكيانات

### 5. التقارير المالية
- تقرير شهري: دخل / صرف / رصيد
- تقرير لكل محفظة ومسار
- تقرير نسب التحصيل
- تقرير المتأخرين

### 6. النزاعات والاعتراضات
- الاعتراضات المعلقة التي تحتاج رد
- النزاعات المفتوحة
- سجل القرارات السابقة

### 7. علاقات الكيانات
- عرض الكيانات المرتبطة
- إنشاء / تعديل / إنهاء علاقة
- عرض التداخلات في العضوية

> لتفاصيل مركز صحة الصندوق، انظر: [fund_health_center.md](fund_health_center.md)
