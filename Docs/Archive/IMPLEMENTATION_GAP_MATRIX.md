## منفذ ومتحقق برمجياً

| المجال | الحالة | الدليل |
|---|---|---|
| فصل Platform عن Tenant في JWT والواجهات | مكتمل | حساب المنصة لا يستطيع استخدام واجهات أعضاء الكيانات، ولكل بوابة رمز دخول وCookie مستقلان |
| طالب الانضمام قبل العضوية | مكتمل | كل الدعوات وطلبات الانضمام تنشئ `MembershipApplication` ولا تنشئ `Membership` |
| قرار قبول أو رفض الطلب | مكتمل | قبول المدير/المؤسس ينشئ العضوية ويسجل إشعاراً وAudit Log، والرفض لا يمنح وصولاً |
| مركز مراجعة العضويات | مكتمل | الطلبات تظهر في لوحة الأعضاء ومركز مراجعة الكيان والمركز العام |
| سجل وصول المنصة | مكتمل | السبب والنطاق إلزاميان، ويصل إشعار للمؤسس والمدير |
| حالات الكيان على مستوى المنصة | مكتمل | `ACTIVE`, `SUSPENDED`, `READ_ONLY`, `PENDING_REVIEW` موجودة في المولد والتحقق |
| تطبيق التعليق خارج مسار الكيان المباشر | مكتمل | الحارس يحل سياق الكيان من المحفظة والمسار والعضوية والقرار والطلب والمستند |
| مولد سنة تشغيلية مترابطة | مكتمل | عضويات واشتراكات ودفعات وقرارات وتصويتات وصرف ووثائق وتنبيهات وسجلات دخول و12 لقطة شهرية |
| واقعية الأعداد | مكتمل | التحقق يفشل إذا ساوى/تجاوز عدد اللجان أو المحافظ عدد أعضاء الكيان أو ساوت لجنة جميع أعضائه |
| تغطية السيناريوهات بأكثر من سجل | مكتمل | كل حالة لطلب العضوية لها سجلان على الأقل، مع دعوات صالحة ومنتهية ومستنفدة وملغاة |
| Rollback | مكتمل | `seed:fresh` يأخذ نسخة ظل ويستعيدها تلقائياً عند الفشل، و`seed:rollback` يعيد أحدث نسخة |

## التحقق التشغيلي

| المجال | الحالة | المطلوب |
|---|---|---|
| تطبيق Migration وزرع القاعدة الفعلية | مكتمل | طُبق `20260622090000_add_membership_applications` وزُرع ملف `medium` بتاريخ 2026-06-22 |
| فحص دخول كل مستخدم فعّال | مكتمل | نجح `81/81` عبر `seed:smoke-logins` مع ظهور سياق الكيانات لكل شخص |
| فحص ظهور البيانات في الواجهات | مكتمل | نجحت اختبارات Playwright للوحة العضو وطلبات العضوية والجوال دون أخطاء console |
| فحص لوحة المنصة | مكتمل | نجح دخول المالك وظهرت الكيانات الست وحالات المنصة الأربع |
| فحص عزل البوابتين | مكتمل | Tenant إلى Platform = `403` وPlatform إلى Tenant = `403` |

## متطلبات Platform Layer الإضافية

| المجال | الأولوية | الملاحظة |
|---|---|---|
| تصدير البيانات أثناء التعليق الكامل | مكتمل | يوجد endpoint تصدير موحد ومستثنى صراحة من حارس التعليق |
| اعتراض مدير الكيان على تعليق المنصة | مكتمل | يوجد نموذج ودورة تقديم وعرض ورد مستقلة عن نزاعات الكيان التشغيلية |
| وصول دعم محدد بزمن وموافقة مسبقة | عالية — غير مكتمل | المكوّن البصري موجود، لكن لا توجد جلسة دعم بموافقة ونطاق ونافذة زمنية قابلة للإلغاء |
| RLS على مستوى Tenant | عالية — جزئي | migration والسياسات موجودة، لكن Tenant context غير مربوط بكل طلب والخدمات لا تستخدم `prisma.extended` فعلياً بعد |

## توافق الواجهات مع فلسفة `Docs/07_Frontend` — تدقيق 2026-06-22

### ما اكتمل

| الشاشة | الحالة | ما تغيّر |
|---|---|---|
| `dashboard` | ✅ مكتمل | نقل نصوص رحلة الانضمام إلى i18n؛ كانت مُكوَّدة بالعربية |
| `analytics` | ✅ مكتمل | أضيف `AccessReasonPanel` للمستخدمين بدون دور رقابي بدلاً من صمت صفحة فارغة |
| `disputes` (قائمة) | ✅ مكتمل | `RuleSummaryPanel` أصبح دائماً في الأعلى بدلاً من كونه مشروطاً بوجود نزاعات |
| `disputes/[id]` | ✅ مكتمل | أضيف `RuleSummaryPanel` قبل نموذج الحل يشرح مسار التسوية |
| `disbursements` | ✅ مكتمل | أضيف `RuleSummaryPanel` يشرح شرط القرار المعتمد عند اختيار المسار |
| `finance` | ✅ مكتمل | `buildPaymentTimeline` مُحوَّل لـ i18n كامل؛ أضيف `RuleSummaryPanel` في تبويب الدفع |
| `subscriptions` | ✅ كانت مكتملة | `RuleSummaryPanel` + كشف التداخل + موافقة الشروط موجودة مسبقاً |
| `rules` | ✅ مكتمل | `AccessReasonPanel` للمستخدمين غير الإداريين؛ أزرار التبويبات مُحوَّلة لـ i18n؛ `RuleSummaryPanel` عند تحديد مسار |
| `wallets/[id]` | ✅ مكتمل | استُبدلت جميع `confirm()` و `alert()` بـ `ConfirmActionDialog` وrسائل inline؛ أضيف قسم السياق الحوكمي |
| `paths/[id]` | ✅ مكتمل (كان BUG) | أضيفت أزرار التبويبات المفقودة كلياً من JSX؛ أضيف header المسار و KPI row و `RuleSummaryPanel` و `VisibilityNotice` لكل بند و `StatusBadge` للاشتراكات |

### ما لا يزال معلقاً (ما وراء MVP)

| المجال | السبب |
|---|---|
| Support View بجلسة زمنية وموافقة مسبقة | يحتاج backend جديد (session model + approval flow) |
| RLS كامل على مستوى قاعدة البيانات | يحتاج ربط TenantContextService بكل طلب |
| واجهة Conflict of Interest الاستباقية | ذُكر في `06` لكن لم يُصمَّم بعد |
