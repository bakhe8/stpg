# معجم المصطلحات (Glossary)

المصطلحات الأساسية المستخدمة في توثيق CollectiveTrustOS، مرتبة أبجدياً بالإنجليزية.

---

| المصطلح بالإنجليزية | المصطلح بالعربية | التعريف |
|---|---|---|
| **Active** | فعّال | حالة العضو عندما يكون متوافقاً مع شروط المسار ويدفع ويستفيد |
| **Appeal** | اعتراض | طلب رسمي من العضو لمراجعة قرار يعتقد أنه غير عادل أو مخالف للائحة |
| **Audit Trail** | سجل التدقيق | سجل شامل غير قابل للحذف يوثق كل تغيير وقرار في النظام |
| **Benefit Scope** | نطاق الاستفادة | مجموعة فرعية من الأعضاء يحق لهم الاستفادة من بند معين |
| **Campaign** | حملة مؤقتة | كيان مؤقت يُنشأ لغرض محدد (مثل: جمع تبرعات لعلاج) وينتهي بانتهاء الغرض |
| **Compatibility Engine** | محرك التوافق | قلب النظام — يقارن شروط العضو مع خصائص المسار ويحدد حقوقه |
| **Conditional** | مشروط | حالة العضو عندما يريد الاشتراك لكنه ينتظر تحقق شروطه |
| **Decision** | قرار | أي قرار رسمي في النظام (صرف، تعديل، إنشاء) يكون كائناً مستقلاً قابلاً للتتبع |
| **Dispute** | نزاع | خلاف رسمي بين عضو والإدارة أو بين أعضاء يحتاج تحكيم |
| **Entity** | كيان | التجمع الأساسي الدائم: عائلة، عمارة، حي، قبيلة |
| **Entity Relationship** | علاقة بين كيانات | الرابط بين كيانين: تداخل عضوية، دعم مالي، محفظة مشتركة، تفويض |
| **Exited** | منسحب | حالة العضو عندما يخرج من المسار |
| **Governance Path** | مسار الحوكمة | طريقة اتخاذ القرار والثقة داخل المحفظة (مجلس إدارة، تصويت، قرار فردي) |
| **Interested** | مهتم | حالة العضو المبدئية عندما يُبدي رغبته في الاشتراك لكنه لم يقرر |
| **Ledger** | الدفتر المالي | نظام محاسبي مزدوج يتتبع كل عملية مالية بدقة |
| **Ledger Segmentation** | تقسيم الدفتر | فصل الأرصدة على مستوى الكيان والمحفظة والمسار والبند |
| **Membership** | عضوية | علاقة الشخص بكيان معين (الشخص قد يملك عدة عضويات) |
| **Member Preferences** | تفضيلات العضو | الشروط التي يضعها العضو للمشاركة (نوع الحوكمة، سقف الصرف) |
| **Modular Monolith** | تطبيق معياري موحد | بنية برمجية: تطبيق واحد لكنه مقسم داخلياً إلى وحدات مستقلة |
| **PBAC** | صلاحيات مبنية على السياسة | Policy-Based Access Control — صلاحيات تعتمد على السياق وليس الدور فقط |
| **Person** | شخص | الإنسان الحقيقي — حساب واحد لكل إنسان في النظام |
| **Policy Versioning** | إصدارات السياسات | حفظ نسخ تاريخية من اللوائح للرجوع إليها عند النزاعات |
| **Relationship Engine** | محرك العلاقات | يدير العلاقات بين الكيانات ويمنع التضارب والتكرار |
| **Rules Engine** | محرك القواعد | يدير القواعد المتغيرة والقابلة للتكوين |
| **Separable Benefit** | مصلحة قابلة للفصل | محفظة استفادتها فردية (علاج، زواج) — تصلح لمسارات متعددة |
| **Shared Benefit** | مصلحة مشتركة | محفظة استفادتها جماعية (حارس، مصعد) — تحتاج حوكمة موحدة |
| **Snapshot** | لقطة | نسخة تاريخية من الشروط واللوائح وقت اشتراك العضو |
| **Spending Item** | بند الصرف | التصنيف النهائي للإنفاق داخل المحفظة (علاج عاجل، صيانة مصعد) |
| **Sub-Entity** | كيان فرعي | تجمع داخل كيان أكبر (فخذ من قبيلة، عائلة داخل صندوق عائلة ممتدة) |
| **Subscription** | اشتراك | التزام مالي دوري يمنح العضو حقوقاً محددة |
| **Subscription State Machine** | آلة حالة الاشتراك | الحالات المتتالية: Interested → Conditional → Active → Suspended → Exited |
| **Supported Only** | داعم فقط | حالة العضو عندما يدفع بدون حق استفادة (تبرع) |
| **Suspended** | معلّق | حالة العضو عندما يتوقف اشتراكه بسبب عدم الدفع أو تغيير الشروط |
| **Wallet** | محفظة | الوعاء المالي الذي يجمع المال لتحقيق هدف واحد محدد |
